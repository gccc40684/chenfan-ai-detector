## 1. 背景与问题

### 当前状态
AI 生成内容（ChatGPT、Claude、Gemini 等）已广泛普及，但市场上缺乏针对中文用户优化的轻量级检测工具。现有产品要么定价过高（Originality.ai $14.95/月起），要么中文支持差，要么广告过多影响体验。

### 核心痛点
1. **教育用户**：学生用 AI 写作业，教师需要快速检测，但 Turnitin 不对个人开放
2. **内容创作者**：需要确保内容原创性，避免被平台判定为 AI 生成
3. **隐私顾虑**：现有免费工具多需上传云端，用户担心敏感内容泄露

### 不做这个的成本
- 目标用户继续使用体验差/价格高的现有工具
- 错失中文 AI 检测市场的窗口期（竞品尚未深度本地化）

---

## 2. 目标与成功指标

### 业务目标
- 验证 AI 检测工具的市场需求
- 建立首批种子用户（目标：MVP 上线 30 天内 1000 次检测）

### 用户目标
- 用户能在 10 秒内完成检测并获得可信结果
- 检测准确率 >= 85%（与 GPTZero 免费版持平）

### 核心 KPI
| 指标 | 定义 | 目标 | 时间 |
|-----|------|------|------|
| 日检测次数 | 每日完成的检测数量 | 100 次/日 | 上线后 30 天 |
| 检测完成率 | 开始检测后完成的比例 | >= 90% | 上线后 14 天 |
| 用户留存率 | 7 日内重复访问比例 | >= 20% | 上线后 30 天 |
| 付费转化率 | 免费用户升级付费比例 | >= 2% | 上线后 60 天 |

### 明确不在范围内 (Out of Scope) - MVP 阶段
- ❌ 自有 AI 检测模型训练
- ❌ 浏览器插件
- ❌ 移动端 App
- ❌ 高级会员功能（如 API 访问、批量检测）

## 3. 用户与场景

### 目标用户

| 用户类型 | 描述 | 核心需求 |
|---------|------|---------|
| **主要：教育工作者** | 教师、助教、论文指导老师 | 快速检测学生作业/论文是否 AI 生成 |
| **次要：内容创作者** | 自媒体、SEO 从业者 | 检测文章原创性，避免平台惩罚 |
| **第三：普通用户** | 对 AI 检测好奇的个人 | 临时检测一段文字，无需注册 |

### 核心用户故事
- 作为教师，我需要上传学生的 Word 文档快速检测 AI 概率，以便判断是否存在学术不端
- 作为内容创作者，我需要检测长篇文章是否会被判定为 AI 生成，以便在发布前修改
- 作为普通用户，我需要在不注册的情况下快速检测一段文字，以满足好奇心

## 4. 需求

### 4.1 功能需求

#### 功能一：文本检测（核心）

**描述：** 用户粘贴或输入文本，系统返回 AI 生成概率

**字数限制策略（按会员等级）：**

| 等级 | 单次字数限制 | 月检测次数 | 价格 |
|-----|-------------|-----------|------|
| 免费版 | 2,000 字 | 50 次 | 免费 |
| 基础版 | 10,000 字 | 200 次 | ¥19/月 |
| 专业版 | 50,000 字 | 无限 | ¥49/月 |
| 团队版 | 100,000 字 | 无限 + API | ¥199/月 |

**交互流程：**
```
1. 用户进入首页
2. 在文本框粘贴/输入文本
3. 实时显示当前字数 / 等级限制
4. 点击"检测"按钮
5. 显示加载状态（预计 3-5 秒）
6. 展示检测结果页面
```

**检测结果展示：**
- AI 生成概率（0-100% 可视化进度条）
- 判定结论（"极可能 AI 生成" / "可能 AI 生成" / "极可能人类撰写"）
- 详细指标：Perplexity 分数、Burstiness 分数
- 可疑段落高亮（如适用）
- 字数统计与升级提示（接近限制时显示）

**边界情况：**
- 文本为空或仅空白字符 → 显示"请输入文本"
- 文本超过当前等级限制 → 显示"文本超出限制，升级会员可检测更多字数" + 升级按钮
- 当月检测次数用完 → 显示"本月检测次数已用完，升级会员继续" + 升级按钮
- 检测超时（10 秒）→ 显示"检测超时，请重试"
- API 错误 → 显示"服务暂时不可用，请稍后重试"

---

#### 功能二：文件上传检测

**描述：** 支持上传多种格式文件进行检测

**支持格式：** .txt, .md, .docx, .pdf

**文件处理逻辑：**
- .txt/.md：直接读取文本内容
- .docx：前端解析文档提取纯文本（使用 mammoth.js）
- .pdf：前端解析提取文本（使用 pdf-parse 或 pdfjs-dist，仅支持文字型 PDF）

**字数统计规则：** 按实际提取出的中文字符 + 英文单词数计算

**交互流程：**
```
1. 用户点击"上传文件"或拖拽文件到指定区域
2. 前端解析文件提取文本
3. 显示提取字数 / 等级限制
4. 如字数超限，提示升级
5. 用户点击检测（后续流程同文本检测）
```

**边界情况：**
- 文件超过 10MB → 显示"文件过大，请上传小于 10MB 的文件"
- 不支持的格式 → 显示"仅支持 .txt, .md, .docx, .pdf 格式"
- PDF 为扫描件（无文字层）→ 显示"无法提取 PDF 文字，请上传文字型 PDF 或直接粘贴文本"
- 文件读取失败 → 显示"文件读取失败，请检查文件后重试"

---

#### 功能三：检测历史（本地存储）

**描述：** 保存最近检测记录到浏览器本地

**功能：**
- 自动保存最近 50 条检测记录
- 显示历史列表（时间、文本前 50 字/文件名、AI 概率）
- 点击历史记录可重新查看详情
- 提供"清空历史"按钮
- 历史记录按会员等级同步（登录后）

**技术方案：** 
- 未登录：localStorage，最多 20 条
- 登录后：Cloudflare KV，最多 50 条/等级

---

#### 功能四：多语言支持

**描述：** 支持中文和英文界面

**要求：**
- 默认根据浏览器语言自动选择
- 提供语言切换按钮
- 检测功能本身支持中英文文本

---

#### 功能五：会员系统（预留框架）

**描述：** MVP 阶段预留会员功能入口，显示"即将推出"

**前端预留：**
- 检测结果页显示当前等级和限制
- 字数接近限制时显示升级提示
- 设置页预留"升级会员"入口

**后端预留：**
- 用户表结构预留会员字段
- API 预留权限校验中间件

### 4.2 非功能需求

| 类别 | 要求 |
|-----|------|
| **性能** | 页面首屏加载 < 2 秒；检测 API 响应 P99 < 8 秒；PDF/docx 解析 < 3 秒 |
| **安全** | 文件仅在前端解析，不上传服务器；API Key 不暴露到前端 |
| **兼容性** | 支持 Chrome 90+, Safari 14+, Firefox 88+, Edge 90+ |
| **可访问性** | 支持键盘导航；满足 WCAG 2.1 AA 基础要求 |
| **分析** | 接入 Cloudflare Web Analytics（匿名化，无 Cookie） |

## 5. 设计

### 页面结构

```
/
├── Hero 区域
│   ├── 标题：AI 内容检测器
│   ├── 副标题：快速检测文本是否由 AI 生成
│   └── 检测输入区
│       ├── 文本框（带字数统计：当前/限制）
│       ├── 拖拽上传区（支持 txt/md/docx/pdf）
│       └── 检测按钮
├── 检测结果区（检测后显示，默认隐藏）
│   ├── 等级提示条（当前等级 + 剩余次数）
│   ├── 总体评分卡片（AI 概率大数字 + 进度条）
│   ├── 详细指标（Perplexity / Burstiness）
│   ├── 文本高亮展示（可疑句子标红）
│   └── 升级提示（接近限制时显示）
├── 功能特点（三栏介绍）
│   ├── 支持多格式（txt/md/docx/pdf）
│   ├── 长文本检测（最高 10 万字）
│   └── 隐私保护（本地解析不上传）
├── 会员等级对比表
└── 页脚

/settings（预留）
├── 语言切换
├── 历史记录管理
└── 升级会员（显示"即将推出"）
```

### 核心设计原则
- **极简**：首页只有一个主要操作（检测）
- **透明**：实时显示字数限制和剩余额度
- **可信**：展示具体指标（Perplexity/Burstiness）增强可信度
- **渐进**：免费用户自然接触到升级场景

## 6. 技术方案

### 架构
```
Cloudflare Pages (静态托管 + Edge Functions)
├── 前端：React + Tailwind CSS
│   ├── 文件解析：mammoth.js (docx) + pdfjs-dist (pdf)
│   └── 字数统计：中文按字符，英文按单词
├── Edge Function：调用 OpenRouter API
└── 存储：
    ├── localStorage (未登录用户)
    └── Cloudflare KV (登录用户，预留)
```

### API 集成
- **提供商：** OpenRouter
- **模型：** `anthropic/claude-3-haiku`（平衡准确率与成本）
- **备用模型：** `openai/gpt-4o-mini`（主模型不可用时切换）

### 检测 Prompt 设计
```javascript
const DETECTION_PROMPT = `分析以下文本，判断其是否由 AI 生成。

请从以下维度分析：
1. Perplexity（困惑度）：文本的可预测性，AI 文本通常更"平滑"
2. Burstiness（突发性）：句子长度和结构的变化程度
3. 语言模式：是否存在 AI 常见的过渡词、套话

输出格式（JSON）：
{
  "ai_probability": 0-100,
  "confidence": "high|medium|low", 
  "suspicious_segments": ["可疑句子1", "可疑句子2"],
  "reasoning": "简要分析说明",
  "metrics": {
    "perplexity_score": 0-100,
    "burstiness_score": 0-100
  }
}

待检测文本：
"""
{{user_text}}
"""
`;
```

## 8. 验收标准 (Acceptance Criteria)

### 功能验收

- [ ] **AC1:** 用户在首页文本框输入任意文本，实时显示字数统计（中文按字符，英文按单词）
- [ ] **AC2:** 免费用户输入超过 2000 字时，显示升级提示并阻止检测
- [ ] **AC3:** 用户可上传 .txt/.md/.docx/.pdf 文件，前端解析后显示提取字数
- [ ] **AC4:** 扫描型 PDF（无文字层）上传后显示友好提示
- [ ] **AC5:** 点击"检测"按钮后，8 秒内显示检测结果
- [ ] **AC6:** 检测结果包含 AI 概率（0-100%）、判定结论、Perplexity 和 Burstiness 分数
- [ ] **AC7:** 当 AI 概率 > 60% 时，文本中高亮显示可疑段落
- [ ] **AC8:** 检测结果页显示当前会员等级、剩余检测次数
- [ ] **AC9:** 每次检测后，记录自动保存到浏览器历史，最多保留 20 条（未登录）
- [ ] **AC10:** 历史记录页面可查看过往检测，点击可重新查看详情
- [ ] **AC11:** 界面支持中英文切换，切换后所有文案即时更新
- [ ] **AC12:** API 错误时显示友好提示，提供"重试"按钮

### 性能验收

- [ ] **AC13:** 首页首屏加载时间 < 2 秒（3G 网络模拟）
- [ ] **AC14:** 10MB 以内 PDF/docx 文件解析时间 < 3 秒
- [ ] **AC15:** 检测 API P99 响应时间 < 8 秒
- [ ] **AC16:** 页面 Lighthouse 性能评分 >= 80

### 兼容性验收

- [ ] **AC17:** 功能在 Chrome 90+, Safari 14+, Firefox 88+ 正常运行
- [ ] **AC18:** 移动端（iOS Safari / Android Chrome）界面正常可用
- [ ] **AC19:** 文件拖拽上传在桌面端浏览器正常工作

## 9. 风险与依赖

| 风险 | 影响 | 概率 | 缓解措施 |
|-----|------|------|---------|
| OpenRouter API 不稳定 | 检测服务中断 | 中 | 实现备用模型切换；添加缓存机制 |
| 检测准确率不达预期 | 用户信任度低 | 中 | Prompt 持续优化；设置用户预期（"仅供参考"） |
| 成本超支 | 项目不可持续 | 低 | 监控 API 调用量；设置每日调用上限 |
| 竞品快速跟进 | 差异化优势消失 | 高 | 快速迭代，建立用户口碑 |
| PDF/docx 解析兼容性 | 部分文件无法读取 | 中 | 提供"直接粘贴文本"备选方案 |
