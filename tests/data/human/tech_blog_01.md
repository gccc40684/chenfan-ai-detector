## 为什么我从 React 迁移到了 Vue

说实话，这个决定并不是一蹴而就的。三年前刚开始做前端的时候，React 简直是业界标杆，不学 React 好像就找不到工作似的。但我用了两年之后，越来越觉得有些别扭。

首先是 JSX。刚开始觉得挺酷的，JavaScript 里写 HTML，多方便啊。但项目一大，那些嵌套层数深到离谱的组件，看着就头大。而且团队成员水平参差不齐，有人写出来的 JSX 简直像意大利面条，理都理不清。

去年接了个新项目，我索性决定试试 Vue。一开始还担心团队不接受，毕竟大家 React 用习惯了。结果？真香！

Vue 的单文件组件简直是强迫症福音。模板、脚本、样式，各归其位，清清楚楚。新来的同事上手也特别快，基本上看两天文档就能干活了。

当然，React 也有它的优势。生态更成熟，大公司用得多，招聘的时候也更容易找到人。但如果你问我现在选哪个？看项目吧。小而快的项目，Vue 真的省心。

不过话说回来，工具只是工具，重要的还是人。我见过用 React 写出屎山代码的，也见过用 jQuery 写出优雅代码的。框架选得再好，写代码的人不行也是白搭。

最近听说 Vue 3 的组合式 API 也挺好用的，等手头这个项目忙完，准备深入研究一下。有已经在用 Composition API 的朋友吗？欢迎交流经验。
